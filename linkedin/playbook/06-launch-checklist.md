# 06 · Launch checklist

> Company-agnostic pre-launch gate. Copy into the tenant's setup guide and tick per tenant. Nothing external goes out until the boxes that matter are green.

## Access & continuity
- [ ] Company Page exists at the canonical `linkedin.com/company/<slug>`
- [ ] **≥2 Super admins** (owner + backup) — single point of failure removed
- [ ] Content admins added for day-to-day operators (if any)
- [ ] (If in scope) founder profile lists the company as current position

## Branding
- [ ] **Logo** (300×300) uploaded, reads at small size, saved
- [ ] **Cover** (1128×191) uploaded, message survives lower-left logo overlap + responsive crop, saved
- [ ] Checked on **desktop and mobile** live views

## Copy & settings
- [ ] Name, Tagline (≤120), About (≤2,000, front-loaded), Website set
- [ ] Industry, Company size, Company type set
- [ ] Founded year decided (set or intentionally blank)
- [ ] HQ location set
- [ ] Custom button chosen + URL wired
- [ ] Up to 3 community hashtags set (brand tag owned)
- [ ] Page verified (if the flow is available) [VERIFY LIVE]
- [ ] ≥1 employee lists the company (People section populates)

## Content
- [ ] **First post** written, approved, and its visual produced
- [ ] Posting model decided: human-in-the-loop **now**; Community Management API pursued in parallel (see [`02`](02-agent-control-api.md))
- [ ] Cadence + first-week/first-month plan drafted (tenant `content-strategy.md`)
- [ ] CTA + destination link confirmed (and UTM-tagged if tracking)

## Compliance
- [ ] No banned terms / off-brand claims on any public field
- [ ] Any human-judgment values signed off (industry, size, founded, button)
- [ ] Nothing violates the brand's "do not say" list

## Go / no-go (outward-facing — the owner's call)
- [ ] Owner (and the brand's public face) approves the copy + first post **before** anything publishes
- [ ] The page is not empty when traffic starts (first post live or scheduled)
