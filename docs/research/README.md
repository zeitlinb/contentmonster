# Research — index & how it's organized

> Durable home for **market/operator/recency research** that informs what we build.
> Findings live here; **decisions** they lead to get promoted to the charter
> (`docs/core-charter/`), and **operational steps** become runbooks (`docs/runbooks/`).
> Raw research is evidence, not doctrine — chase anything load-bearing to a primary source.

## Layout

```
docs/research/
  README.md                ← this index
  book-publishing/         ← the book realm (charter D1)
    YYYY-MM-DD-<topic>.md   ← dated artifacts, one per research pass
```

Naming: `YYYY-MM-DD-<source>-<topic>.md` (e.g. `2026-06-22-last30days-ai-authority-book-creators.md`).
Each artifact carries frontmatter (`source`, `date`, `query`, `status`) + a synthesis + the
raw evidence (handles/URLs/engagement) so claims stay verifiable.

## Sources we file here

- **last30days pulses** — Reddit/X/YouTube/web recency (Brad-invoked `/last30days`). Value = recency, not depth.
- **Workflow research** — multi-agent sweeps (operator teardowns, KDP mechanics, length/price, domain demand).
- **Manual/web** — ad-hoc deep dives.
